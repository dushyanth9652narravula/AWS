# Elastic Compute Cloud (EC2)

## Introduction

- Amazon EC2 stands for Elastic Compute Cloud. It is the fundamental service within the Amazon Web Services that provides on demand, scalable, secure and virutalized computing capacity (esentially vitual servers) in the cloud.

- EC2 enables users to run applications without the need of the up front hardware investments,allowing for rapid scaling of compute resources within minutes rather than hours or days.

- If you breakdown the name EC2, then we have three words. Those are 

  **Elastic** : Elastic refers to the ability to automatically scale the computing resources such as increasing or decreasing the no of servers based on demand, traffic or workload requirements.

  **Compute** : Compute provides the computing power (CPU), Memory (RAM), Storage and networking capacity to run applications.

  **Cloud** : Cloud tells that all these virtual servers are hosted in Amazon global data centers allowing access across all over the world.

- So overall, EC2 allows users to rent virtual machines that acts as servers. These servers can be configured to host websites, run applications, process data or support backend databases. The Core functionalities of EC2 is :

  **Virutal Servers** : Users can launch virtual machines running different operating systems such as linux, windows or mac os

  **Complete Control** : Users can have complete adminstrative access over the instances so that they can install any software , configure networking and manage security settings.

  **Auto Scaling** : EC2 can automatically add or remove instances to match traffic ensuring high availability and cost efficiency.

  **Flexible Storage** : EC2 integrates with EBS (Elastic Block Storage) for persistent, high performance storage which acts as an virtual hard drives for the instances.

  **Security** : EC2 provides security groups to act as virtual firewalls by restricting the incoming and outgoing traffic.

## Configuring the EC2 Instance

- To Create an EC2 instance first we need to search for EC2 in the console and then we need to click on Launch instance button under instance section. Once you click on launch instance button, it prompt you to ask the following things.

- At first, you have to name the instance. Once you names that instance then it will ask you to select the AMI.

- AMI stands for Amazon Machine Image which is an image that provides software such as Operating System, Application Software, and storage settings required to setup and boot an EC2 instance. Once you selected the required AMI for your instance then it asked you to select the type of instance you willing to have.

- Instance Type actually provides you the computing power required to run your application which means it provide the CPU and Memory for you server.

- After selecting the type of instance required to run your application, then we have to create a key pair which is required to access this EC2 instance through command line. Since AWS doesn't provide any graphical interface for that instance, so we need to access that server via command line interface only. So inorder access the server via command line we need the key pairs to login to that instance. We have two type of key pairs there. Once is for putty and other is for SSH. Putty is used for windows version lessthan or equal to 8. If your windows version is more than 8 then it recommended to use SSH.

- Once we have created the key pairs, now we need to configure the network seetings. Network settings include setting up Network Inteface Card which we actually need to access this EC2 instance via internet and then setting up the firewalls which actually controls the incoming and outgoing traffic. By default AWS provides a virtual network interface card provides a way to the EC2 instance to access the internet. To configure the firewall setting, we need to create a security group and add the rules that controls incoming and outgoing traffic and then we have to acctach that security group to these instance.

- Once we configured network settings, now we have to configure the storage for this EC2 instance. This storage acts as an virtual hard drive to this EC2 instance. AWS provides different types of storages such as EBS, EFS, and EC2 instance storage etc.

- Once we have configured all these settings, we have one more thing to configure which is EC2 UserData which is optional. EC2 User Data is used to bootstrap our EC2 instances. Bootstrapping means launching command when machine starts. That script runs only once at the instance first start. EC2 User Data is used to automate the boot tasks such as Installing Updates, Softwares, Downloading common files from internet, etc. But EC2 User Data runs with root user of the instance.

## EC2 Instance Types

- AWS offers different types of EC2  instances that are optimized for different use cases. Each instances offers unique combinations of compute, memory and storage to power your specific workload needs. The major types of EC2 instances are General Purpose, Compute Optimized, Memory Optimized, Storage Optimized, Accelerated Computing and HPC Optimized.

- Before going to know in detail about these instances, let see the naming comvention of AWS EC2 instances. For example comsider this instance i.e `m5.2xlarge`. Here `m` tells us the instance class. `5` tells us the generation of that instance (as AWS imporves them over the time). `2xlarge` indicates the size of the instance. The size of instance starts from small and goes upto xlarge, 2xlarge and so on. As instance size increase computing resources such as CPU, Memory, Storage and Networking resources increases.

- **General Purpose Instances** : General purpose Instances provides a good balance of Compute, memory and netorking resources and can be used for for many workloads. These instances are good for many applications such as web servers, code repositories and small to medium databases. Most common instances in these class are T class instances such as t2.micro, t3.micro, M8g and so on.

- **Compute Optimized Instances** : Compute Optimized Instances are ideal for Compute bound applications that benifit from high performance processors. Some example workloads for compute instances are batch processing, media transcoding and dedicated gaming servers. Most common instances of these type are C class such as c6gn, c6g, c5, c5a, c5n etc. 

- **Memory Optimized Instances** : Memory Optimized Instances are designed to deliver fast performances for workloads that process large datasets in memory. For example these instances are good for High performance relational / non relational databases, Distributed Web scale cache stores, In-memory databases optimized for Businees intelligence, applications performing real time processing of big unstructured data. Most common instances of these type are r6g, r5, r5a  and x class instances

- **Storage Optimized Instances** : Storage Optimized Instances deliver millions of low latency, random I/O operations per second to applications. These are designed for workloads that require high sequential read and write access to large datasets in local storage. For example these instances are good for high frequency online transaction processing systems (OLTP), Relational & NoSQL databases, Data Warehouse Applications, Distributed File Systems Cache for in memory databases auch as Redis etc. Most common instances of these type are I8g, I8ge, I7i etc.

- **Accelerated Computing** : Accelerated Computing Instances use hardware accelerators or co processors to perform functions more efficiently. For example they can perform floating point number calculations, graphics processing, or data pattern matching. Some instances of these type are P6e instance etc. 

- **HPC Optimized Instances** : High performace computing instances offer best price performance for running HPC workloads at scale. HPC instances are ideal for applications that benifit from high-performance processors such as complex simulations, deep learning, and visual effects rendering. Example for these instances are Hpc8a etc.

## Security Groups 

- Security Groups are the fundamental of network security in AWS. They actually acts as virtual firewalls to AWS EC2 instances that controls incoming and outgoing traffic. Security groups contain inbound rules which controls all the incoming traffic. Security group has oubound rules which actually contolls all ougoing traffic. But by default an instance has access to any thing in the internet. So by defualt the destination for outbound rules is all.

- While creating inbound rules, we majorly configure two things. One is ports and other one is authorized IP addresses such as IPv4 or IPv6. Some of the classic ports are `22` - Used for SSH(Secure Shell) i.e used to login linux instance, `21` - FTP (File Tranfer Protocol) i.e used for uploading files into file share, `22` - SFTP (Secure File Tranfer Protocol) i.e used to upload file using SSH, `80` - HTTP (Hyper Text Transfer Protocol) i.e used to access unsecured websites, `443` - HTTPS (Hyper Text Transfer Protocol Secure) i.e used to access secured websites, `3389` - RDP (Remote Desktop Protocol) i.e used to login to the windows instances. So once we configure the port and IP address then the instance having configured IP can access the instances that is attached to this security group via configured port.

- As we know security groups rules can be referenced by IP or anothor security group. If two EC2 instances in the AWS wants to communicate then we usually configure inbound rules such that they are referenced by other security groups instead of IPs. This is because IP's of EC2 instances changes when you stop and re-start an instance. 

- Some good things we know about security groups are :

  1. Security Groups can be attached to multiple instances. They are locked down to a region / VPC combination. They actually live outside of EC2, so if traffic is blocked then EC2 instance won't see it.

  2. Its good to maintain one seperate security group for SSH access. If your application is not accessible (time out), then its security group issue. If your application gives a connection refused error then its application error or its not launched which ports might be closed or applications are not running on those ports.

  3. All inbound traffic is blocked by default and all outbound traffic ia authorized by default.
